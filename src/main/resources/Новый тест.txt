Каждый раз, когда будем писать новый тест, необходимо первый раз запустить кнопкой слева, а затем, вверху открыть
Edit Configurations... и вставить переменную в стоку VM options через пробел -Dtestng.dtd.http=true
После этого нужно запустить тест кнопкой вверху справа.

Чтобы покрыть юни-тестами каждое решение, необходимо обернуть в виде своего метода, который можно вызвать и результат решения
должен возвращаться методом.

Тестовые методы возвращают значение void, т.е. ничего не возвращают.

Нужно разносить тесты. Один тест - одна проверка.

Различают анотации тестовые и хуки (hook - крючок), см. презентацию.

Параметризация с помощью data provider - запуск теста несколько раз с различными наборами данных.
Позволяет не дублировать код и прогонять тест много раз.

Параметризация с помощью xml-файла - позволяет динамически прикладывать значения при запуске.
